## Summary

A targeted, bounded A4 proof-event scan was added to `runtime_evidence` so rare A4 proof rows remain visible even when high-volume monitor/`live_executor` audit rows push them outside the generic audit tail. The generic tail stays at 50; a separate bounded window (default 5000 lines) is scanned **only** for attributed A4 proof events, and proof/stability are built from the merged, de-duplicated reachable proof rows.

Core principle: **Evidence that exists but is unreachable is not runtime truth.**

## Scope

Evidence visibility only. No proof re-run, no RPC/Helius call, no global tail expansion, no auto-promotion, no readiness claim.

## What Changed

- **Generic audit tail unchanged** — `DEFAULT_AUDIT_TAIL_LIMIT` remains `50`; all non-A4 evidence collection is untouched and lightweight.

- **Targeted A4 proof scan added** — `readA4ProofScan()` reads a larger bounded window and keeps ONLY rows with `producer === "a4_rpc_proof"` and `eventType === "A4_READ_ONLY_RPC_PROOF"`. It uses an injectable reader (`options.a4ProofScanReader`) for deterministic tests and defaults to a bounded JSONL tail read.

- **Scan limit** — `DEFAULT_A4_PROOF_SCAN_LIMIT = 5000`, overridable per-call via `options.a4ProofScanLimit` (for tests/diagnostics). **Not** `.env`-configurable in this first pass.

- **Proof-row filtering** — non-proof rows are dropped and never exposed; the existing `extractA4ProofFacts` allowlist/sanitizer still rejects any forbidden/secret-shaped row (`secretExposureRisk` flagged without exposure).

- **Dedupe behavior** — `mergeA4ProofRows(genericRows, scanRows)` merges by a stable secret-free key (`timestamp | producer | eventType | proofStatus | method`); a proof present in both windows counts once.

- **Fail-safe behavior** — any scan error degrades to `proofScan = { available: false, limit, errorCode }` with an enumerated code (`A4_PROOF_SCAN_UNAVAILABLE` / `_READ_ERROR` / `_PARSE_ERROR` / `_UNKNOWN_ERROR`); no throw into runtime health, no raw error/path/line content. Proof/stability then fall back to whatever the generic tail supports (`stabilityCandidate` stays `false` unless generic evidence supports it).

- **runtime_health** — `classifyA4Health` now returns a safe `proofScan` summary (`available`, `limit`, `errorCode` allowlisted). Scan-backed `stabilityCandidate: true` still maps via the existing A4.18 rule to `A4_STABILITY_PROOF_OBSERVED` (warning, blockerActive, live/soak readiness false). Blockers (`NOT_CONFIGURED`, `REFUSAL_ACTIVE`, `FALLBACK_DETECTED`, `PROOF_FAILED`, `PROOF_STALE`) still win.

- **dashboard** — display-only `proofScan { available, limit, errorCode }` through a positive allowlist; non-enumerated error codes collapse to `null`; no endpoint triggers proof calls.

- **Tests added** — 16 fixture-only tests (10 evidence, 4 health, 2 dashboard).
