## 5. Dry-Run / Paper / No-Capital Safety Assessment

### 5.1 Scanner refresh — **safe in dry-run/paper mode**

- `scanner_gmgn_trending.js` performs GMGN/DexScreener discovery and appends paper trades; **no on-chain submission**.
- External dependency: `gmgn-cli` and GMGN API credentials (env — **not inspected in this gate**).
- Operational note: may append to `paper_trades.json` and `pipeline_candidates.jsonl` when candidates pass filters — expected paper-mode behavior, not capital exposure.

### 5.2 Executor heartbeat refresh — **safe when config is PIPELINE_DRY_RUN**

Preconditions (read from `live_config.json`, not `.env`):

- `executionMode: PIPELINE_DRY_RUN`
- `dryRunMode: true`
- `liveArmed: false` (expected from `--status`)

Safety properties (from `live_executor.js` / `OPERATIONS.md`):

- `PIPELINE_DRY_RUN` uses identity-only signer; no signing or submission.
- `--loop` in LIVE mode blocked unless `FOMO_ALLOW_LOOP_LIVE=YES` (not required for current config).
- Lock file records `mode`, `dryRunMode`, `liveArmed` each refresh — visible evidence of posture.

**Caution:** Starting `--loop` runs pipeline observation cycles and appends audit rows. This is operational activity but **not** live trading under current config. Human should run `node live_executor.js --status` immediately before any future start.

### 5.3 Monitor (PID 6568) — **safe, observational**

- `monitor.js` manages paper positions; may invoke `liveExecutor.executeLiveExit` only when an **open live position** exists (none currently).
- Does **not** produce scanner health or executor lock evidence.

### 5.4 Dashboard (PID 34068) — **read-only consumer**

- `/api/runtime-health` re-reads evidence files on each request; no write side effects.
