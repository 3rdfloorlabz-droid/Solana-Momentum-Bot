## Summary

Read-only RPC proof results can now be represented in runtime evidence and
runtime health through safe, attributed audit events. When a fresh, secret-safe,
non-fallback `READ_ONLY_RPC_OK` proof has been recorded by a future approved
runner, runtime health reflects `A4_READ_ONLY_RPC_VERIFIED`. This
implementation ran no proof, called no RPC/Helius, and added no proof-triggering
endpoint.

## Scope

Runtime visibility only. No execution authority, no live readiness, no proof
execution. Attribution before trust.

## What Changed

### Proof event shape recognized

Attributed audit event (produced only by a future approved runner; NOT written
by this change):

- `producer`: `a4_rpc_proof`
- `eventType`: `A4_READ_ONLY_RPC_PROOF`
- `invocationContext`: `a4_read_only_rpc_proof`
- `payload` (allowlist only): `proofStatus`, `providerLabel`, `endpointClass`,
  `method`, `slotObserved`, `slotValuePresent`, `latencyMsBucket`,
  `publicFallbackUsed`, `secretSafe`, `errorCode`

Forbidden anywhere in the event (rejected by construction / by the reader):
endpoint, url, uri, RPC URL, API key, headers, request body, raw slot value,
wallet, signature, transaction, full error object, stack trace, `process.env`.

`a4_rpc_proof.js` gained two pure helpers:
- `sanitizeA4RpcProofForAudit(proofResult)` — positive allowlist; drops every
  unknown/forbidden key by construction.
- `buildA4RpcProofAuditEvent(proofResult, options)` — assembles the canonical
  event object (still not written here).

### runtime_evidence reads newest safe proof event

- Reads the newest row where `producer === "a4_rpc_proof"` and
  `eventType === "A4_READ_ONLY_RPC_PROOF"`.
- Ignores wrong producer, wrong eventType, malformed rows.
- Rejects any row carrying a forbidden/secret-shaped field or a secret-like
  string; flags `secretExposureRisk: "proof_event_forbidden_field_detected"`
  without echoing the offending value.
- Every field passes an allowlist; unknown values collapse to safe sentinels.
- Exposes additive `evidence.a4Evidence.proof` with `proofStatus`,
  `providerLabel`, `endpointClass`, `method`, `slotObserved`, `slotValuePresent`,
  `latencyMsBucket`, `publicFallbackUsed`, `secretSafe`, `errorCode`,
  `proofObservedAt`, `freshness`.
- Freshness threshold: 24 hours (`fresh` / `stale` / `unknown`).

Guarded posture elevation (does not weaken config/refusal/fallback detection):
- Proof can only change posture when the config-derived status is
  `A4_CONFIGURED_UNVERIFIED`.
- Fresh + `READ_ONLY_RPC_OK` + `secretSafe` + not public-fallback →
  `A4_READ_ONLY_RPC_VERIFIED`.
- `READ_ONLY_RPC_FAILED` → `A4_PROOF_FAILED`.
- Stale successful proof → `A4_PROOF_STALE`.
- `NOT_CONFIGURED` / `REFUSAL_ACTIVE` / `FALLBACK_DETECTED` are never promoted by
  a proof event; a proof present without config evidence is surfaced but never
  elevates posture.

### runtime_health maps proof evidence

New statuses in `A4_HEALTH_STATUS` / `A4_WARNINGS` / `A4_STATUS_MAP`:
- `A4_READ_ONLY_RPC_VERIFIED` — severity `warning`, `blockerActive: true`,
  reason: one read-only proof observed; stability + human approval still
  required.
- `A4_PROOF_FAILED` — severity `blocked`, `blockerActive: true`.
- `A4_PROOF_STALE` — severity `warning` (conservative), `blockerActive: true`.

`classifyA4Health` now surfaces an allowlisted `proof` summary. Invariants held:
`supportsLiveReadiness` and `supportsSoakClaim` are hard-`false` for every A4
status, and A4 never relaxes an existing runtime-health verdict.

### Dashboard display changes

Additive, display-only:
- New statuses added to display allowlists, recommended actions, and operator
  messages (fail-closed / not-live-ready wording).
- `sanitizeA4ProofForDisplay` exposes only `proofStatus`, `method`,
  `providerLabel`, `endpointClass`, `freshness`, `latencyMsBucket`,
  `publicFallbackUsed`, `secretSafe`, `slotObserved`, `slotValuePresent`.
- No dashboard endpoint triggers a proof call; no control-route coupling.

### Tests added

16 fixture-only tests across the four A4 test suites (see Test Summary). All use
synthetic audit rows / in-memory evidence; none call RPC, read `.env`, load a
signer, or start a server.

## Status Boundary

`A4_READ_ONLY_RPC_VERIFIED` means one fresh, attributed, secret-safe read-only
proof exists.

It does not mean:
- A4 resolved
- live readiness
- strategy validity
- scanner freshness
- capital readiness
