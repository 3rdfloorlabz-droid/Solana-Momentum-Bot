## What Changed

- **`proofStability` summary added** to `a4Evidence` (additive; `proof` unchanged). Shape:
  ```
  {
    successCount, freshSuccessCount,
    firstProofAt, latestProofAt,
    separationBucket: "<15m" | ">=15m" | "unknown",
    providerConsistent, endpointClassConsistent,
    providerLabel, endpointClass,
    fallbackObserved, failureObserved, secretSafe,
    withinFreshnessWindow, stabilityCandidate,
    threshold: { minSuccesses, minSeparationMs, freshnessMs }
  }
  ```
  Allowlisted fields only — no raw endpoint/URL/API key/headers/raw slot/wallet/signature/transaction/`process.env`/raw error/stack trace.

- **Threshold implemented** (first-pass, from A4.15/A4.17; NOT `.env`-configurable):
  - `minSuccesses = 2`
  - `minSeparationMs = 15 * 60 * 1000` (15 minutes)
  - `freshnessMs = 24 * 60 * 60 * 1000` (24 hours)

  `stabilityCandidate` is `true` only when: ≥2 fresh `READ_ONLY_RPC_OK` successes; ≥15 min apart; both within the 24h window; consistent `providerLabel`; consistent `endpointClass`; `endpointClass === "dedicated"`; `publicFallbackUsed === false` on all considered rows; `secretSafe === true` on all considered rows; no failure observed; no fallback observed; no secret-like/forbidden field detected.

- **Event selection:** only rows with `producer === "a4_rpc_proof"` and `eventType === "A4_READ_ONLY_RPC_PROOF"` are considered (reusing the existing `extractA4ProofFacts` sanitizer). Wrong producer/type, malformed rows, and secret-shaped rows are ignored; a secret-shaped row flags `secretSafe:false` without exposing anything.

- **runtime_health mapping:** when base posture is `A4_READ_ONLY_RPC_VERIFIED` and `stabilityCandidate === true`, status becomes `A4_STABILITY_PROOF_OBSERVED` — `severity: warning`, `blockerActive: true`, `supportsLiveReadiness: false`, `supportsSoakClaim: false`, `recommendedAction: "Record explicit human approval decision before any A4_VERIFIED_DEDICATED consideration."` Stability never overrides `NOT_CONFIGURED`, `REFUSAL_ACTIVE`, `FALLBACK_DETECTED`, `PROOF_FAILED`, or `PROOF_STALE`.

- **dashboard display updates:** additive, display-only stability summary through a positive allowlist. No dashboard endpoint triggers proof calls.

- **Tests added:** 22 fixture-only tests (12 evidence, 8 health, 2 dashboard).
